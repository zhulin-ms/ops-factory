# cluster kubernetes
生产环境运行在 3 个 Kubernetes 集群上：cn-east（主）、cn-south（灾备）、cn-north（边缘）
集群版本统一为 K8s 1.28，容器运行时 containerd 1.7
每个集群约 120 个节点，核心业务 Pod 副本数不低于 3

# networking
内部服务通信使用 Istio service mesh，版本 1.20
入口流量经 Nginx Ingress → SLB，TLS 在 SLB 终止
跨集群通信走专线 + WireGuard VPN，延迟 < 5ms

# database
主数据库为 MySQL 8.0 集群（InnoDB），一主两从，半同步复制
Redis 6.x 哨兵模式用于缓存和会话存储，3 主 3 从
MongoDB 6.0 副本集用于日志和非结构化数据存储

# monitoring
监控体系：Prometheus + Thanos（长期存储）+ Grafana
日志：Filebeat → Kafka → Elasticsearch → Kibana
链路追踪：OpenTelemetry Collector → Jaeger
告警通道：Alertmanager → 飞书机器人 / PagerDuty（P0 级）

# cicd
CI/CD 流水线：GitLab CI → Harbor 镜像仓库 → ArgoCD 部署
制品管理：Harbor（镜像）+ Nexus（Maven/npm）
分支策略：main → release → hotfix，合并需双人 review
