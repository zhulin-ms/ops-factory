# naming
Kubernetes 资源命名：{service}-{component}-{env}，如 order-api-prod
Helm chart values 文件按环境区分：values-dev.yaml / values-staging.yaml / values-prod.yaml
告警规则命名：{severity}_{service}_{metric}，如 critical_order_api_error_rate

# scripts
运维脚本统一放在 ops-scripts 仓库，按 {category}/{script-name}.sh 组织
脚本必须有 --dry-run 模式，批量操作必须有确认提示
Cron 任务统一用 CronJob 管理，禁止在宿主机上直接 crontab

# documentation
每个服务在 Confluence 上维护一份运维手册，包含：架构图、依赖、启停命令、常见问题
重大变更前写 RFC 文档，包含：背景、方案、影响分析、回滚计划、验证步骤
故障复盘使用统一模板：时间线 → 根因 → 影响 → 改进措施 → 责任人

# sla
核心服务可用性目标 99.95%（月度），非核心 99.9%
API 响应时间 P99 < 500ms，P95 < 200ms
数据库慢查询阈值 500ms，超过需优化或加索引
